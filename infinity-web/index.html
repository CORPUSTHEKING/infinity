Documentation Architecture Breakdown

Comprehensive analysis of variables, functions, stylings, and usage guidelines captured inside the srcdir.md file.

The codebase represents a sophisticated Gatsby/React documentation site (likely for npm, given the branding and text) utilizing @primer/react for its design system, framer-motion for animations, and MDX for content rendering.

1. Core UI Components (./components/)

These files handle the structural and interactive user interface elements of the site. They primarily use CSS Modules (.module.css) for scoped styling, with occasional styled-components.

FileVariables & FunctionsStylingUsage / How to Use
breadcrumbs.jsBreadcrumbs(), BreadcrumbItem({item, path}).PrimerBreadcrumbs (margin).Render at the top of a page layout. Auto-generates navigation trails using usePage() and getNav.
header.jsHeader(), NpmHeaderBarstyled-components for gradients, CSS modules for sticky positioning and flexbox layouts.Place at the top of the app tree. Provides desktop/mobile navigation and search inputs.
link.jsgetLocalPath(href), Link(), LinkNoUnderline(), GatsbyLinkWithoutSxProps()Uses clsx to conditionally apply .showUnderline.Use as a drop-in replacement for <a> or <Link>. Automatically handles internal Gatsby routing vs. external URLs.
nav-drawer.jsNavDrawer(), Drawer({isOpen, onDismiss})Fixed positioning, full-screen overlays, scrollable dark-themed inner boxes.Mobile slide-out menu. Handles its own open/close state and traps focus using react-focus-on.
nav-items.jsNavItem(), NavItems(), ExternalNavItem(), Navigation()Font-sizing based on depth (.NavList_Item_depth1, etc.), hidden on desktop using media queries.Renders hierarchical sidebar links. Pass items (from getNav) to build nested accordions.
page-footer.jsformat(date), pluralize(), Contributors(), PageFooter()Flexbox grids, user avatar grids, borders, and tooltip margins.Render at the bottom of markdown pages. Auto-pulls editUrl and GitHub commit data from page context.
search.jsSearchResults(), Desktop(), Mobile()Complex z-indexing, absolute overlays, absolute widths, and Framer Motion logic.Use <Search.Desktop> or <Search.Mobile> in headers. Takes props injected by the useSearch hook.
sidebar.jsusePersistentScroll(id), Sidebar()Sticky positioning, fixed width (270px), calc(100vh - header).Place next to the main content area. Retains scroll position across page loads via sessionStorage.
site-title.jsNpmLogo({size}), SiteTitle({logo})Flexbox alignment, SVG polygon colors (#cb3837).Renders the site name and optional npm logo. Clickable link to /.
skip-nav.jsSkipLink, SkipBox, SkipNavstyled-components. Visually hidden via transform until focus.Accessibility feature. Place near the top of the DOM to allow keyboard users to skip to main content/search.
table-of-contents.jsTableOfContentsItems(), TableOfContents(), withTableOfContents(), Mobile(), Desktop()Sticky desktop sidebars, rounded bordered boxes, hidden/shown via media queries.Generates jump-links for headers. Renders differently based on viewport.
text-input.jsTextInputstyled-components overriding @primer/react. Forces 16px.Use for any text inputs (like search) to ensure mobile compatibility.
variant-select.jsVariantItem(), useVariantFocus(), VariantMenu(), useVariants(), VariantSelect()100% width on mobile, auto on desktop.Use on pages that have multiple versions (e.g., CLI v8 vs v9). Builds a dropdown to switch paths.
visually-hidden.jsVisuallyHiddenclip: rect(0,0,0,0), 1px height/width.Wrap text meant strictly for screen readers.

2. Layouts, Pages, & Root Files

These files form the structural foundation and entry points of the application.

FileVariables & FunctionsStylingUsage / How to Use
layout.jsContainer(), HeroLayout(), DefaultLayout(), Layout()Max-widths, dynamic padding, CSS Grid/Flexbox for sidebars.Wraps all MDX content. Automatically switches between layouts.
page.jsGlobalStyles, PageElement()CSS Variables, global body styling, flex layout.Gatsby wrapPageElement. Injects global CSS, SkipNav, Header, Sidebar.
root.jsRootElement()N/AGatsby wrapRootElement. Wraps the entire app in MDXProvider and ThemeProvider.
head.jsbuildTitle(), Head(), PassThrough()N/AGatsby Head API. Generates SEO metadata.

Custom React hooks encapsulating logic for routing, layout states, and search.

FileVariables & FunctionsReturnsUsage / How to Use
use-breakpoint.jsgetMatches(), useMediaQuery(query), useBreakpoint(bp, minMax), useIsMobile()booleanUse to conditionally render React components based on viewport.
use-location-change.jsuseLocationChange(){ change: bool, current: string, previous: string }Use to trigger side effects when the user navigates to a new page.
use-page.jsPageProvider, usePage()Context objectCall anywhere inside a page to access the current markdown file's metadata.
use-search.jsuseSearchData(), useCliVersion(), useSearchCombobox(), useSearch()Combobox stateSpawns a Web Worker, sends it GraphQL node data, and manages the Downshift UI state.
use-site-metadata.jsuseSiteMetadata()ObjectCall to retrieve global config data defined in gatsby-config.js.

4. Markdown/MDX Overrides (./mdx/)

These files intercept standard Markdown elements (like # or ```) and render them using custom React/Primer components.

FileVariables & FunctionsStylingUsage / How to Use
code.jsClipboardCopy(), InlineCode, CodeBlock(), Code()Prism.js syntax highlighting, absolute positioned copy buttons.Used automatically by MDX for code blocks.
components.jsHeadings.wrap(), Blockquote, DescriptionList, Table, Note, Screenshot, YouTube, DataTablestyled-components. Re-styles native HTML tags with Primer design tokens.Exported directly to index.js. Use explicitly in MDX files.
nav-hierarchy.jsHierarchyItem(), Hierarchy(), NavHierarchy()Simple list indentation and sub-text font sizing.Use <Index depth={2} /> in MDX files to auto-generate a list of child pages.
index.jsRe-exports all components mapped to HTML tags (h1, p, a, pre).N/AInjected into the MDXProvider in root.js.

5. Utilities, Configurations, & Constants

Helper functions and static values supporting the rest of the application.

FileVariables & FunctionsConcept / StylingUsage / How to Use
constants.jsHEADER_HEIGHT, Z_INDEX, CLI_PATH, SCROLL_MARGIN_TOPHardcoded integers / strings.Import anywhere to maintain visual consistency.
shared.jssharedUses MDX components (<Screenshot>).Import into MDX files to reuse standardized instructions across pages.
theme.jsnpmTheme, ThemeProvider, Theme, LightTheme, DarkThemePrimer theme deeply merged with npm specific colors.Wrap sections of the UI to force specific color palettes contextually.
util/aria-live.jsannounce(message), setContainerContent(), createNoticeContainer()Creates an invisible DOM node (clip: rect(0,0,0,0)).Call announce("Loading complete") to imperatively force screen readers to read a message aloud.
util/get-nav.jsgetItemBreadcrumbs(), getVariantRoot(), getItem(), getHierarchy(), isActiveUrl()Pure logic functions parsing nav.yml.Used extensively by sidebars, breadcrumbs, and variant selectors.
util/omit.jsomit(obj, ...keys)Pure logic function.Use to strip specific props before passing them down to DOM elements.
util/search.worker.jsFuse class, onmessage listener.Web worker logic using fuse.js.Runs in a separate thread. Send it { items } to index, and { query } to search.
